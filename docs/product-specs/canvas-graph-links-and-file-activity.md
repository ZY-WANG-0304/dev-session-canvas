# 画布关系连线与文件活动视图规格

当前状态：已确认。2026-04-21 已按本文规格完成通用连线、provider 文件活动投影、文件节点 / 文件列表节点的 `card` / `minimal` 双风格收口，以及文件对象投影总开关。当前实现以正式开发质量推进，而不是一次性原型；其中 Agent 文件活动事件必须来自 provider 原生结构化事件，不能依赖 PTY 输出解析。第一轮自动文件活动覆盖 `Claude Code` 与仓库内 `fake-agent-provider` 验证路径；`Codex` 因缺少已确认的 provider 原生文件事件接口，当前只保留 no-op 适配，不把未确认能力写成已支持。本轮已通过 `npm run typecheck` 与 `npm run test:webview`；完整 trusted smoke 在本次执行中仍受既有 `verifyLiveRuntimeReloadPreservesUpdatedTerminalScrollbackHistory` 超时阻塞，因此未把其写成本轮已确认验证结论。

## 1. 用户问题

当前画布已经能同时放置多个 `Agent`、`Terminal` 和 `Note`，但“对象之间的关系”与“Agent 正在碰哪些文件”仍然几乎全靠用户脑补：

- 用户看得到节点，却看不清节点之间的依赖、引用或协作关系。
- 用户很难在画布上直接标注“谁依赖谁”“这条线代表什么”。
- 当 Agent 开始读写代码时，用户仍要切回终端输出、编辑器 tab 或文件树，才能追踪当前代码上下文。
- 多个 Agent 共享同一批文件时，关系会迅速变得混乱，用户缺少一个能直接回跳到 VSCode 编辑区的文件视图。

## 2. 目标用户

当前规格继续优先服务已经把 Dev Session Canvas 当作多 Agent 协作主工作面的 VSCode 开发者。用户需要的是：

- 在同一张画布里把对象关系显式化；
- 在不离开画布的前提下理解 Agent 与代码文件的关联；
- 在需要深入处理文件时，能一跳回到 VSCode 原生编辑器。

## 3. 核心用户流程

1. 用户把鼠标悬停到任意节点边缘，看到四个固定方向锚点。
2. 用户从一个锚点拖到另一个节点，创建一条通用连线。
3. 用户选中这条连线后，在连线上方看到一个轻量编辑台，可切换箭头模式、颜色、进入标签编辑、重新连接连线两端或删除连线；双击连线时，会直接在连线上原位编辑标签；连线既可以连接其他节点，也可以回指当前节点自身；这些设置会随画布一起持久化。
4. 当 `Agent` 通过 provider 原生结构化事件上报文件读写时，画布会自动出现对应的文件对象视图。
5. 默认模式下，单个文件显示为独立小节点；用户可把展示模式切换成文件列表节点。
6. 用户可通过设置 `devSessionCanvas.fileNode.displayStyle` 在 `card`（卡片风格）和 `minimal`（极简风格）之间切换；默认值为 `minimal`。
7. 用户可通过设置 `devSessionCanvas.files.enabled` 一键关闭整个文件活动功能域；默认值为开启。该设置与 `devSessionCanvas.runtimePersistence.enabled` 一样，需要重新加载窗口后才生效。关闭后，系统不再接收或保留 `fileReferences`，画布不再提供文件节点、文件列表节点、自动文件关系线和文件过滤入口。
8. 在 `minimal` 风格下，文件节点会收口成贴内容边框的紧凑对象；文件列表节点会收口成接近 VSCode Source Control Changes 的单行文件列表，并在节点头部提供 `列表视图 / 树形视图` 切换。
9. 用户点击文件节点，或点击文件列表节点中的文件条目，VSCode 会在编辑区打开对应文件；若画布当前承载在编辑区，宿主会复用或创建相邻 editor group 打开目标文件，而不是覆盖画布所在组。
10. 当多个 Agent 共享同一文件时，文件节点模式下保留一个共享文件节点并连到多个 Agent；文件列表模式下会额外生成共享文件列表节点。
11. 用户删除某个 Agent 节点后，系统会移除该 Agent 的文件引用；只有仍被其他 Agent 引用的文件对象才继续保留。

## 4. 在范围内

- 节点通用连线：
  - 悬停节点边缘显示四向锚点：上、右、下、左。
  - 用户可通过拖拽锚点创建节点到节点的连线。
  - 连线允许从节点连回当前节点自身。
  - 连线支持三种箭头模式：无箭头 / 单向箭头 / 双向箭头。
  - 连线支持 Obsidian 风格的颜色能力：默认颜色加 6 个预设颜色。
  - 连线支持文本标签。
  - 选中任意连线后，会在连线上方显示 VSCode Workbench 风格的轻量编辑台；属性编辑统一走这条工具条。
  - 轻量编辑台与标签区域必须分离定位，避免工具条和标签 / 标签编辑框相互挤占。
  - 轻量编辑台使用当前仓库已在使用的 VSCode `codicon`，提供箭头模式切换、颜色切换、编辑标签和删除连线四个入口。
  - 标签默认以贴线文本显示；双击连线可直接进入标签原位编辑；编辑态显示独立轻输入框；未命名连线在未编辑时不显示占位标签。
  - 选中连线后可从连线两端拖动，重新调整源节点 / 目标节点与锚点；端点重接热区默认保持透明，不额外显示显眼圆形把手。
  - 选中连线后按 `Delete` 键可删除。
  - 默认连线颜色与“拖拽中尚未落点”的预览连线保持一致；选中态通过额外 outline 与端点 handles 区分，不通过主线换色区分。
  - 连线数据作为画布权威状态的一部分持久化。
- 文件活动基础设施：
  - 宿主侧定义 provider 文件活动事件接口，按 provider 适配，不依赖 PTY 输出文本解析。
  - `Claude Code` 使用官方 hooks 路线把 `Read` / `Edit` / `Write` 工具事件转换为结构化文件活动。
  - 仓库内 `fake-agent-provider` 提供同形态测试事件，确保自动化验证可重复。
  - `Codex` 当前没有已确认的 provider 原生文件事件接口，因此本轮只保留 no-op 适配；不把“自动文件活动”验收写到 Codex 路径上。
  - 支持全局配置 `devSessionCanvas.files.enabled`：
    - `true`：在当前窗口启用文件活动功能域；支持的 provider 会记录 `fileReferences`，并按当前展示模式投影文件节点或文件列表节点，同时显示对应自动关系线。
    - `false`：该设置在 reload window 后生效；生效后宿主不再启动文件活动接线，不再保留 `fileReferences`，并停用文件节点、文件列表节点、自动文件关系线与 sidebar 文件过滤入口。
- 文件节点模式：
  - 默认展示模式为文件节点。
  - 同一路径的文件节点在整张画布中唯一；多个 Agent 共用同一文件时共享该节点。
  - 文件节点会根据 Agent 的文件活动自动创建，并自动生成与 Agent 的关系线。
  - 文件节点支持点击后在 VSCode 编辑区打开文件；若画布当前在编辑区承载面，打开动作必须落到独立 editor group。
  - 文件节点支持自动布局与用户手动拖拽。
  - 文件节点图标当前只提供有限的“常见扩展名 -> `codicon`”映射，并在其他情况回退到通用文件图标；本版本不把“完整复用当前 VSCode File Icon Theme”写成已实现。
  - 文件节点支持显示风格配置 `devSessionCanvas.fileNode.displayStyle`：
    - `card`：保留当前卡片式边框、阴影和内边距；当没有额外路径信息时，不再用 `N 个 Agent 引用` 填充第二行。
    - `minimal`：默认值；移除阴影和多余装饰，边框紧贴内容，自适应当前图标 / 路径显示内容的尺寸；文本模式下只保留首行主标签，不额外显示第二行路径补充。
  - 文件节点不引入文件专属 hover 遮罩；hover 继续复用画布节点的统一选中态与锚点反馈。
  - 文件节点支持显示模式配置：
    - 文件类型图标 + 文件路径（默认）
    - 只显示文件类型图标
    - 只显示文件路径
  - 文件路径支持两种显示口径：
    - 只显示文件名（默认）
    - 显示相对所属 workspace folder 根目录的路径；若当前是多根 workspace，则在前方补上 workspace folder 名称前缀以避免同名路径冲突；若不在 workspace 内，则显示规范化绝对路径
- 文件列表节点模式：
  - 用户可通过配置切换到文件列表节点模式。
  - 每个 Agent 会获得自己的文件列表节点，用图标和尾部读写标识标明每个文件是读、写还是读写。
  - 若文件被多个 Agent 共享，则系统会额外生成共享文件列表节点，并从相关 Agent 自动连线过去。
  - 点击文件列表中的文件条目，会在 VSCode 编辑区打开文件；若画布当前在编辑区承载面，打开动作必须落到独立 editor group。
  - 文件列表节点同样受 `devSessionCanvas.fileNode.displayStyle` 控制：
    - `card`：保留当前卡片式文件列表节点。
    - `minimal`：改成接近 VSCode Changes 的单行文件列表；每个文件一行，显示文件图标、文件名以及尾部彩色 `R` / `W` 标识。
  - 文件列表中的文件条目交互遵循 VSCode list token 语义：
    - hover 使用 `list.hoverBackground` / `list.hoverForeground`。
    - 当前活动中的已选中文件使用 `list.activeSelectionBackground` / `list.activeSelectionForeground`。
    - 当焦点移到其他节点或离开当前 Webview / VSCode 区域时，已选中文件保持选中，但切换到 `list.inactiveSelectionBackground` / `list.inactiveSelectionForeground`。
  - 文件列表条目的选中态属于节点内部局部 UI 状态，按节点 ID 持久化在 Webview state 中，不回写宿主权威状态。
  - `minimal` 风格下，文件列表节点头部提供 `列表视图` 和 `树形视图` 两个切换入口：
    - `列表视图`：平铺所有文件条目。
    - `树形视图`：按目录结构分组显示文件条目。
- 过滤与生命周期：
  - 在 `devSessionCanvas.files.enabled = true` 时，支持 `include` / `exclude` glob 过滤文件对象投影；过滤控件位于 sidebar 的 `常用操作` section 中，直接显示 `包含文件` / `排除文件` 两个输入框。
  - 过滤编辑在 sidebar 内就地完成，不再通过单独菜单或弹出输入框完成。
  - `include` / `exclude` 只影响文件节点 / 文件列表节点 / 自动边的显示投影，不修改 `fileReferences` 权威状态。
  - 当 `devSessionCanvas.files.enabled = false` 且已完成 reload 后，文件过滤入口不可用，也不会保留上一轮文件活动状态。
  - 删除 Agent 节点后，系统移除该 Agent 的文件引用；没有剩余引用的文件节点 / 文件列表节点自动清理。

## 5. 不在范围内

- 通过 PTY 输出、终端文本或自然语言摘要猜测文件读写。
- 让用户在第一轮手工创建文件节点或文件列表节点。
- 给连线预设业务语义、强制关系类型或关系校验。
- 对 `Codex` 提供未经确认的伪结构化文件活动接线。
- 完整白板级自动布局、分组折叠或关系线自动整理。

## 6. 关键对象与状态

### 通用连线

- 连线 ID
- 源节点 / 目标节点
- 源锚点 / 目标锚点
- 箭头模式
- 颜色
- 标签文本
- 是否为自动维护的关系线

### 文件活动引用

- 规范化文件路径
- 最近一次活动时间
- 引用该文件的 Agent 集合
- 每个 Agent 对该文件的访问方向：读 / 写 / 读写

### 文件节点模式

- 文件节点位置与尺寸
- 文件图标展示信息
- 文件路径显示文案
- 与哪些 Agent 存在自动关系线

### 文件列表节点模式

- Agent 专属列表节点内容
- 共享文件列表节点内容
- 每个文件条目的访问方向图标
- 文件列表当前内部视图：`list` 或 `tree`
- 列表节点位置与尺寸

## 7. 验收标准

- 用户把鼠标移到节点边缘时，可以看到四向锚点，并能通过拖拽成功创建连线。
- 用户创建的连线可以通过选中态轻量编辑台切换无箭头 / 单向箭头 / 双向箭头、切换颜色、进入标签编辑、重新连接两端和删除，并在 reload 后保持。
- 用户可以创建或重接一条指向自身的自环连线，并在 reload 后保持。
- 双击连线时，标签会在连线位置原位编辑，而不是弹出宿主输入框或显示无法编辑的占位胶囊。
- 默认连线和拖拽中的预览连线使用同一默认灰色 token；选中态不通过主线换色来区分。
- 连线切换到预设颜色后，仍可通过“默认颜色”恢复到默认灰色 token。
- 选中连线后按 `Delete` 键会删除该连线，而不会误删节点。
- `Claude Code` Agent 在触发文件读写工具后，画布会实时出现对应文件对象视图；该路径不依赖终端输出文本解析。
- 修改 `devSessionCanvas.files.enabled` 后，当前窗口要在 reload 后才切换到新的文件功能状态；当 `true -> false` 时，下次加载会清空 `fileReferences`、文件节点 / 文件列表节点、自动文件活动边与文件过滤入口；当 `false -> true` 时，只恢复功能可用性，不恢复之前已清空的文件活动状态。
- 默认文件节点模式下，同一路径只保留一个文件节点；多个 Agent 引用同一文件时，该文件节点会自动连到多个 Agent。
- 文件活动派生的连线与手工连线在 UI 和交互入口上没有额外区分；用户修改自动边后，系统会持久化对应覆盖结果，而不是下一次重建立即回退。
- 文件节点可点击并在 VSCode 编辑区打开对应文件；当画布位于编辑区时，目标文件会在独立 editor group 中打开。
- 把 `devSessionCanvas.fileNode.displayStyle` 从 `card` 切换到 `minimal`，或从 `minimal` 切回 `card` 后，文件节点 / 文件列表节点的视觉风格会切换，但位置与连线关系保持稳定。
- `minimal` 风格下，文件节点默认只保留紧贴内容的边框，不再出现卡片式阴影和宽内边距。
- 文件节点在 hover 时不会额外出现文件专属遮罩；交互反馈继续依赖节点统一选中态与锚点显隐。
- 切换到文件列表节点模式后，系统会把 Agent 文件活动收口为列表节点；多个 Agent 共享的文件会单独进入共享文件列表节点。
- `minimal` 风格下，文件列表中的每个文件条目都能以单行明确标出彩色 `R` / `W` 关系，并支持点击打开。
- `minimal` 风格下，文件列表节点支持 `列表视图 / 树形视图` 切换，两种视图都保持相同的打开文件行为。
- 文件列表条目在 hover / active selected / inactive selected 三态下分别使用 VSCode list 对应 token，而不是统一复用 `focusBorder` 蒙层。
- 在文件功能开启时，`include` / `exclude` sidebar 过滤会以常用操作区中的内嵌输入框形态影响文件对象与自动边的显示结果，但不会删改 `fileReferences`；在文件功能关闭并 reload 后，这组入口不会继续显示为可用状态。
- 删除 Agent 节点后，如果某文件不再被任何 Agent 引用，则对应文件节点 / 文件列表节点会被自动清理；若仍被其他 Agent 引用，则继续保留。
- `Codex` 在当前版本不会凭空出现“自动文件活动已支持”的假象；没有事件就不生成自动文件对象。

## 8. 开放问题

- `Codex` 的 provider 原生文件活动事件接口何时具备，以及届时应通过 CLI 参数、插件还是其他结构化接口接入。
- 对高度定制的 File Icon Theme，第一轮是否需要覆盖全部字体图标与衍生目录图标规则，还是允许先覆盖常见文件图标主路径。
