# 画布外层控件侧栏化规格

当前状态：草案。本文档用于收口 `功能体验.md` 第 2 条，并定义“画布只保留底角必要控件，非空间控件迁到 VSCode 侧栏”的产品范围与验收口径。

## 1. 用户问题

当前画布已经去掉了右侧选中节点信息区，也把左下角遮挡问题修掉了，但整体仍然保留了左上角 hero 和右上角操作 panel。这比之前更克制，却仍然有四个明显问题：

- 固定 panel 继续占用画布本体，让空间化工作面带着 dashboard 感。
- 创建对象、恢复状态、更新时间和说明性文字大多不依赖节点空间关系，却仍然压在画布里。
- 当编辑器宽度较窄，或画布里同时存在多个运行窗口时，顶部固定 panel 会持续抢占可用空间。
- 用户缺少一个位于 VSCode 宿主层、可以稳定访问的极简全局入口，只能把很多“应该离开画布”的动作继续留在画布里。

## 2. 目标用户

本规格优先服务已经在 VSCode 中使用 OpenCove 画布进行多 Agent 协作的开发者。用户需要一张尽量干净的主工作面来理解节点、关系和整体进度，同时又需要一个不打断画布的轻量入口来完成全局动作。

## 3. 核心用户流程

1. 用户在 VSCode 中打开 OpenCove 的侧栏入口，并看到与当前 workspace 绑定的最小必要状态。
2. 用户从侧栏打开或重新定位画布。
3. 用户在需要新增对象时，通过侧栏触发创建，而不是依赖画布右上角固定 panel。
4. 用户回到画布后，画布顶部不再显示固定说明卡片或操作区，只保留节点本体和底角导航元素。
5. 当 workspace 未受信任、画布尚未打开或有执行型对象在运行时，用户仍能在侧栏看到简短状态，并理解当前哪些动作可用、哪些动作已降级。

## 4. 在范围内

- 移除画布 Webview 中左上角与右上角的常驻说明/操作 panel。
- 画布内默认只保留两类常驻角落 UI：
  - 左下角：导航性质控件，例如缩放和回到全局视图。
  - 右下角：全局定位性质控件，例如 MiniMap。
- 默认提供一个 VSCode 侧栏承载面，作为以下能力的主入口：
  - 打开或定位画布
  - 创建 `Agent`、`Terminal`、`Task`、`Note`
  - 重置宿主状态
  - 展示最小必要状态，例如 workspace trust、画布是否可定位、节点总数，以及必要的执行状态摘要
- 当侧栏不可见时，上述动作仍可通过命令入口到达。
- 侧栏整体信息密度保持极简，不引入长段说明、教学大卡片或复制节点正文。

## 5. 不在范围内

- 在侧栏复刻选中节点详情、Agent/Terminal 连续输出、Task/Note 正文或其他 richer metadata
- 用侧栏取代画布成为主要工作面
- 新增多个侧栏 view、tab 或 dashboard 式信息墙
- 在侧栏中维护完整节点列表、关系图或 mini inspector
- 在本轮重新设计节点本体字段、执行会话窗口或对象关系模型

## 6. 关键对象与状态

### 侧栏承载面

- 画布入口状态：未打开、可定位、不可用
- workspace trust 状态
- 当前节点总数
- 当前执行型对象的最小摘要，例如运行中数量或受限提示
- 允许创建的对象类型集合

### 画布 Webview

- 节点对象图
- 当前视口与相机状态
- 左下角导航控件
- 右下角全局定位控件
- 临时错误提示或短时反馈

### 共享动作

- 打开或定位画布
- 创建对象
- 重置宿主状态
- 在受限模式下禁用或降级执行型动作

## 7. 验收标准

- 用户打开画布后，左上角和右上角不再出现常驻 hero、统计卡片或创建/恢复 panel。
- 左下角只保留导航性质控件，右下角只保留全局定位性质控件；这两个区域都不承载长段说明文本。
- 用户可以从默认侧栏入口打开或定位画布，并完成创建对象与重置宿主状态。
- 侧栏不展示选中节点的重复信息，也不复制节点正文、会话 transcript 或连续输出。
- 当 workspace 未受信任时，侧栏会禁用、隐藏或降级执行型对象入口，并给出明确说明。
- 在窄编辑器宽度或多运行窗口场景下，画布顶部可用空间明显大于当前“顶角 panel”方案，不再被固定大块 UI 持续占用。
- 当用户折叠、移动或暂时离开侧栏时，关键动作仍可通过命令入口完成，不会因为侧栏不可见而失去主路径。

## 8. 开放问题

- 侧栏中的“创建对象”第一版应收敛为单一创建入口加对象类型选择，还是保留多个直接入口。
- 右下角全局定位控件第一版是否始终保留 MiniMap，还是在节点很少或编辑器极窄时允许自动收起。
- “运行中执行单元数”是否属于第一版真正需要展示的最小状态，还是只保留节点总数与 trust 状态即可。
