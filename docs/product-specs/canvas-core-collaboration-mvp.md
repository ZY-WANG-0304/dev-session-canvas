# 画布核心协作 MVP 规格

## 1. 用户问题

在 VSCode 中并行使用多个 Agent、终端、任务和笔记时，开发者的上下文容易分散在多个 panel、editor、标签页和临时记录里，导致以下问题：

- 很难快速看清当前有哪些执行单元正在活动
- 很难理解终端、任务、笔记和 Agent 之间的关系
- 切到 VSCode 原生终端或编辑器后，容易丢失回到全局视角的路径
- 在 reload、重开窗口或远程工作区场景下，关键上下文容易断裂

## 2. 目标用户

当前规格优先服务已经在 VSCode 中进行 AI 协作或 Harness Engineering 的开发者。用户通常已经习惯 VSCode 原生终端、编辑器和 workspace 语义，但缺少一个能把多执行单元放回同一空间中理解的主视图。

## 3. 核心用户流程

1. 用户在一个 VSCode workspace 中打开画布。
2. 用户看到一张可平移、缩放的无限画布。
3. 用户可以在画布上创建四类对象：Agent、终端、任务、笔记。
4. 用户可以直接在执行型对象节点内进行最小交互，而不必始终依赖外部 inspector。
5. 用户可以查看这些对象的标题、位置、状态与基本关系。
6. 当用户需要进入终端或其他 VSCode 原生能力时，可以从画布对象跳转过去。
7. 当用户完成局部操作后，可以明确地回到画布，并继续从全局视角理解当前进度。
8. 当 Webview 隐藏、reload 或窗口重开后，系统至少能恢复关键上下文，而不是回到完全空白状态。

## 4. 在范围内

- 一个运行在 VSCode Editor Group 中的主画布入口
- 画布的基础导航能力：平移、缩放、查看对象
- 四类核心对象的最小创建与展示能力：
  - Agent
  - 终端
  - 任务
  - 笔记
- 对象的基础状态展示，例如标题、类型、位置、最近状态摘要
- Agent 对象的最小真实 backend 主路径：
  - 用户可选择 `Codex` 或 `Claude Code`
  - 用户可直接在 Agent 节点内输入消息并发起真实 CLI Agent 运行
  - 节点至少保留连续转录、运行中、完成、失败或停止状态
- 终端对象与 VSCode 原生终端之间的基本跳转与回流
- Task 对象的最小真实编辑主路径：
  - 用户可在节点内编辑标题、状态、任务描述和负责人
- Note 对象的最小真实编辑主路径：
  - 用户可在节点内编辑标题和正文
- 画布关键状态恢复：
  - 至少包含对象图恢复
  - 至少包含最后一次打开画布的基本上下文恢复
- 在未信任 workspace 或能力不可用时，给出明确退化反馈

## 5. 不在范围内

- 独立 app 的 workspace 管理、启动器和多 workspace 组织
- 嵌入式全功能终端
- provider 原生 TUI 的像素级复刻
- 多人实时协作
- 复杂分组、嵌套、自动布局和高级白板能力
- 完整 Agent backend 生态或多 provider 深度接入
- 导入导出格式、分享机制和跨项目同步
- 像素级复刻参考产品视觉与交互

## 6. 关键对象与状态

### 画布

- 打开状态
- 当前对象集合
- 相机位置与缩放
- 当前选中对象

### Agent 对象

- 标题
- 状态
- 节点内的连续转录
- 最近一步摘要
- 最近一次真实运行结果摘要
- 与其他对象的引用关系
- 节点内输入区与运行控制
- 当前绑定的 CLI provider

### 终端对象

- 标题
- 关联的宿主终端会话引用
- 活跃状态摘要
- 从画布跳转到终端、再回到画布的入口

### 任务对象

- 标题
- 状态
- 负责人或关联 Agent

### 笔记对象

- 标题
- 内容摘要
- 与其他对象的引用关系

### 恢复相关状态

- 与 workspace 绑定的对象图和关键上下文
- Webview 局部 UI 状态与宿主权威状态的边界
- 在 reload 或重开窗口后，哪些状态应恢复、哪些允许作为临时态丢失

## 7. 验收标准

- 用户可以从 VSCode 中打开主画布，并在同一工作面中看到无限画布。
- 用户可以创建 Agent、终端、任务、笔记四类对象，并看到它们出现在画布上。
- 用户可以在多对象同时存在时识别每个对象的基本状态，而不必逐个打开独立 panel 才理解当前情况。
- 用户可以直接在 Agent 节点内发送消息并看到连续转录，而不必依赖右侧详情区才能与 Agent 交互。
- 用户可以从终端对象跳转到对应的 VSCode 原生终端，并能清楚知道如何回到画布。
- 当 Webview 被隐藏、重新显示或 reload 后，画布不会无声丢失所有对象状态。
- 当窗口重开或扩展重启后，系统至少能恢复关键上下文，而不是回到全新空白页。
- 在 Restricted Mode 或相关能力不可用时，执行型操作会被禁用或退化，并给出明确反馈。

## 8. 开放问题

- 终端对象的“最近输出摘要”在第一版应做到什么粒度，才能既有价值又不引入过高实现复杂度。
- 第一版是否需要显式连接线，还是先依靠对象内引用信息表达关系。
- 关键上下文恢复的最小集合应以什么口径界定，才能既满足可靠性要求，又不把所有临时 UI 状态都强行持久化。
