# 画布空状态、辅助面板密度与新增节点摆放设计

## 1. 背景

当前画布原型已经具备四类节点、节点内主交互和宿主权威状态主线，但最近一轮实际体验反馈暴露出四个明显问题：

1. 左下角“当前验证范围”浮层遮挡了 React Flow 自带的缩放控件区域。
2. “恢复默认状态”后仍会自动生成一个 `Task` 和一个 `Note`，与“回到初始空画布”的直觉不一致。
3. 右侧“选中节点概况”与节点本体展示的信息高度重复，尤其是 `Task` / `Note`，浪费横向空间。
4. 新增节点仍按固定序号网格摆放，面对宽高差异明显的节点窗口时容易发生遮挡。

这些问题都属于用户直接可见的画布交互，不应继续依赖实现细节隐式决定。

## 2. 问题定义

本轮需要明确的不是视觉风格，而是以下交互规则：

1. 画布的默认状态和“重置”语义到底是什么。
2. 画布外层辅助面板应该保留哪些信息，哪些信息必须让位于画布导航与节点本体。
3. 右侧固定面板是否需要继续承担任何“选中节点信息”职责，还是应该只保留创建与恢复操作。
4. 新增节点应如何放置，才能默认不遮挡已有节点，并尽量出现在用户当前关注区域附近。

## 3. 目标

- 把“空画布”收敛为合法且默认的起始状态。
- 让画布缩放、MiniMap 和主导航控件不再被固定浮层遮挡。
- 去掉右侧固定“选中节点信息”区域，让节点本体成为唯一的主要信息承载面。
- 让新建节点默认落在视口附近，并且不与已有节点发生初始重叠。

## 4. 非目标

- 不在本轮引入完整自动布局、分组排版或关系线整理能力。
- 不在本轮实现可拖拽、可折叠或可停靠的复杂面板系统。
- 不在本轮解决用户手动拖拽后导致的所有碰撞；本轮只保证“新建时默认不重叠”。

## 5. 候选方案

### 5.1 默认保留种子节点

特点：

- 打开或重置后自动出现占位 `Task` / `Note`。
- 用已有节点告诉用户画布里能放什么对象。

不选原因：

- 这会让“重置”变成“重建示例数据”，而不是“回到空白工作面”。
- 当前右上角已经有明确创建入口，不再需要靠种子节点承担教学责任。

### 5.2 继续保留底部验证说明浮层

特点：

- 验证范围信息始终可见。

不选原因：

- 它直接与画布导航控件争夺左下角稀缺区域。
- 验证说明不是一等主交互，不应覆盖缩放与定位能力。

### 5.3 继续保留右侧选中节点信息区

特点：

- 选中节点后，在右侧固定区域展示正文、最近输出或字段详情。

不选原因：

- 这会重新强化“主要信息在节点外”的错误心智。
- `Task` / `Note` 已经是节点内可编辑对象，再重复正文只会增加信息噪音。

### 5.4 新建节点继续使用固定序号网格

特点：

- 节点位置只由创建顺序决定。
- 实现简单，不需要考虑当前视口或节点尺寸。

不选原因：

- 当前节点尺寸差异已经很大，固定间距会让宽窗口互相遮挡。
- 当用户把画布移动到其他区域时，固定网格也无法贴近当前工作上下文。

## 6. 当前结论

### 6.1 空画布是默认态，也是重置目标

- 宿主默认状态改为空节点集合。
- “重置宿主状态”应回到空画布，而不是生成示例 `Task` / `Note`。
- 画布教学责任由上方辅助面板承担，而不是靠宿主塞入假数据。

### 6.2 画布外层面板必须让位于导航控件

- 左下角不再放置固定浮层。
- 原“当前验证范围”信息并入左上角 hero 面板，作为补充说明而不是独立占位块。
- React Flow 左下角控件区域保持无覆盖，确保缩放与回到全局的操作始终可用。

### 6.3 右侧区域不再展示选中节点信息

- 右侧固定区域只保留“创建对象”和“恢复链路说明”。
- 所有与选中节点相关的信息都直接留在节点本体，不再额外复制到右侧。
- 这样可以进一步降低固定面板的信息密度，避免再次长回 inspector。

### 6.4 新建节点采用“视口锚点 + 避碰搜索”

- Webview 在发送“创建节点”消息时，同时发送当前视口中心附近的流坐标作为首选锚点。
- 宿主仍然是位置权威来源；宿主根据节点类型的默认窗口尺寸做矩形碰撞检测。
- 若首选位置与现有节点冲突，则宿主按固定步长向外搜索候选位置，直到找到第一个不重叠的位置。
- 如果 Webview 尚未拿到视口信息，则宿主退回到基于默认锚点的同一套避碰搜索，而不是退回固定序号网格。

## 7. 风险与取舍

- 取舍：右侧保留固定操作入口，但直接删除选中节点信息区。
  原因：连“补充信息”都会继续占用横向空间，而且节点本体已经足够承载当前需要暴露的信息。

- 风险：节点默认尺寸只是估算值，避碰搜索不等于完整布局系统。
  当前缓解：只把它作为“首次创建不重叠”的保护层；用户后续拖拽位置仍保持完全手动。

- 风险：空画布可能让首次进入的用户感到更空。
  当前缓解：在 hero 面板和创建区里明确告知四类对象入口，不再用种子节点代替引导。

## 8. 验证方法

至少需要完成以下验证：

1. 重置宿主状态后，画布回到空节点集合。
2. 左下角缩放控件可直接点击，不再被说明面板遮挡。
3. 右侧固定区域中不再出现任何选中节点信息。
4. 连续新增不同类型节点时，默认位置不会遮挡已有节点。
5. 当画布已平移到其他区域时，新节点会优先落在当前视口附近，而不是总回到旧网格原点。
