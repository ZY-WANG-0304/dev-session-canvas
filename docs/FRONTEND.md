# FRONTEND

本文件用于约束用户可见层文档应记录哪些信息。当前不预填具体界面方案，只保留当前项目特别需要描述的维度。

## 建议记录维度

- 画布入口与打开方式
- 画布的平移、缩放和导航行为
- Agent、终端、任务、笔记四类核心对象的视觉表达
- 对象之间的关系、连接和空间组织方式
- 多对象并行时的视觉层级与信息密度控制
- 选中态、活动态、错误态和完成态的反馈
- 键盘操作、快捷键和可访问性要求
- 小屏幕或窄编辑器宽度下的退化策略

## 使用原则

- 如果界面方案尚未确定，先写问题、约束和候选方向，不要先写具体 UI 结论。
- 如果只是提出方向，明确标注为“候选方案”或“草案”。
- 任何视觉方案都应先回答“它如何帮助用户看清全局”，再讨论风格。
- 如果后续形成稳定的画布交互规范，再单独整理到具体设计文档中。
