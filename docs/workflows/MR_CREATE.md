# MR_CREATE

这个文件只定义 MR 描述应包含的信息和默认格式。

## 目的

- 让 reviewer 和后来者能快速理解这次变更要解决什么问题
- 保证 MR 描述与实际代码、文档和验证证据一致

## 描述内容

MR 描述默认至少包含以下内容：

- 背景 / 目标
- 主要改动
- 验证方式
- 残余风险

## 默认格式

```md
## 背景 / 目标

## 主要改动

## 验证方式

## 残余风险
```

## 写法要求

- 只写当前 MR 实际包含的范围
- 未确认内容不要写成已确认结论
- 验证方式只写实际执行过的内容
- 残余风险只写已知且当前不阻塞合并的问题
