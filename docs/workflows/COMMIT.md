# COMMIT

这个文件定义提交约定。

## 目的

- 统一提交信息格式
- 保持提交说明与实际变更一致
- 避免把无关改动混入当前提交

## 与 MR 的关系

- 本文件只定义 commit 约定；MR 描述见 `docs/workflows/MR_CREATE.md`。
- 如果准备把当前提交推进成 MR，commit 主题应与 MR 的最终目标保持一致。

## 提交前检查

- 只暂存与当前任务直接相关的文件。
- 如果工作区存在无关改动，不要默认把它们一起提交。
- 如果用户指定了提交格式、主题或正文要求，以用户要求为准。
- 如果文档、代码和验证方式之间存在不一致，先收口再提交。

## 默认提交格式

默认使用以下格式：

```text
type(scope): 中文主题
```

## 类型

- `feat`
- `fix`
- `docs`
- `style`
- `refactor`
- `test`
- `chore`
- `other`
- `revert`

## 提交正文

如果需要正文，默认使用中文编号列表说明主要变更：

```text
1. 说明第一项关键变更。
2. 说明第二项关键变更。
```

要求：

- 只写当前提交实际包含的内容。
- 以变更结果为主，不写空泛描述。
- 如果本次提交只包含单一且清晰的改动，可以不写正文。

## 范围选择

- 文档体系调整优先使用 `docs(...)`
- 某个功能或模块变更优先使用对应模块名
- 纯工作流或协作约定调整，可使用 `workflow`
