# RELIABILITY

这个文件用于记录可靠性要求，但当前不预设具体系统方案。

## 建议记录维度

- 画布对象状态是否会无声丢失
- Webview 与宿主之间的状态同步是否可恢复
- Webview 隐藏、reload、窗口重开后，哪些状态必须恢复，哪些允许丢失
- 终端、Agent 或辅助 Note 节点异常时是否可见
- 终端会话、Agent 运行时与画布对象之间的映射是否可恢复
- workspace 重载或扩展重启后是否能恢复关键上下文
- Remote / Codespaces 场景下，宿主与 Webview 分离后状态链路是否仍然可靠
- 外部依赖或 AI 服务失败时如何降级

## 使用原则

- 先定义可靠性目标，再写实现手段。
- 先明确失败类型，再讨论恢复策略。
- 先区分“宿主权威状态”和“Webview 临时状态”，再定义各自的恢复要求。
- 任何依赖远程工作区、终端会话或外部执行 backend 的能力，都应单独说明断连、重试和重连后的表现。
- 如果某个可靠性结论来自实验，应标注来源和证据。
